## Research

### HD Generation Retry Pattern

**Decision**: Return `{ status: "anchor_pending", retryAfterMs: 3000 }` when anchor is not ready.
**Rationale**: Client already has retry logic in `invokeEdgeFunctionWithRetry.ts`; adding a
semantic pending status allows the client to distinguish "anchor not ready" from "error".
**Alternatives considered**: Polling RPC (adds complexity, same result); blocking in-function
(hits Deno 60s timeout for slow anchors).

### CORS Restriction for Admin Functions

**Decision**: Read `SITE_URL` env var; if missing, use `"null"` as origin (deny-all) and log
a warning.
**Rationale**: `SITE_URL` is already a Supabase platform secret available to all edge functions.
**Alternatives considered**: Hardcode `https://snap-shot.ai` — rejected because it breaks
staging environments.

### WASM Lazy Load

**Decision**: Move `import ImageScript from "..."` to inside the function branch that needs
image encoding; Deno supports dynamic `await import()`.
**Rationale**: Deno 1.x supports top-level dynamic import. The WASM module is ~500KB;
skipping it on `create_job` and `get_results` actions saves 200–400ms cold-start.
**Alternatives considered**: Pre-warming (not available on Supabase hosted functions).

### Named Handler Extraction

**Decision**: Extract three handlers in `generate-image/index.ts` into named async functions
in the same file: `handleCreateJob`, `handleWork`, `handleGetResults`. Leave the file
boundary unchanged for now (full module split is Phase 4+ work).
**Rationale**: Minimal diff, easiest to review, lowest regression risk.

### Admin Pagination

**Decision**: Add `limit` and `offset` parameters to the `admin_get_users` and `admin_get_jobs`
RPCs via a new migration. Frontend passes current page * pageSize as offset.
**Rationale**: Simplest SQL approach; compatible with the existing RPC call pattern.
**Alternatives considered**: Cursor-based pagination — better for large tables but requires
UI rework; deferred to a future phase.

### Stripe Unknown Price ID

**Decision**: `throw new Error(`Unknown price ID: ${priceId}`)` which causes a Supabase Edge
Function 500 response and logs the event. Stripe will retry; the team will be alerted via
Supabase logs.
**Rationale**: A 500 causes Stripe to retry the webhook, ensuring the event is not silently
dropped while alerting the team.

## Data Model

### Changes to Existing Tables

**`generation_jobs`** — no schema changes in Phase 3 (config fields locked in Phase 1).

**`generation_tasks`** — no schema changes in Phase 3 (result_url now stores paths — Phase 2).

**`profiles`** — no schema changes. `is_admin` flag already exists. The application-level
change is removing the hardcoded email bypass.

### New Migration: `20260512_admin_pagination_rpcs.sql`

Adds `p_limit INT DEFAULT 50` and `p_offset INT DEFAULT 0` parameters to:
- `admin_get_users(p_limit, p_offset)` — returns users page
- `admin_get_jobs(p_limit, p_offset)` — returns jobs page
- Both RPCs return a total count field alongside the rows for pagination UI.

### CORS Helper (Shared Utility)

A new shared file `supabase/functions/_shared/cors.ts` exports:

```typescript
export function buildCorsHeaders(): HeadersInit {
  const siteUrl = Deno.env.get("SITE_URL");
  if (!siteUrl) {
    console.warn("[CORS] SITE_URL not set — defaulting to deny-all origin");
  }
  return {
    "Access-Control-Allow-Origin": siteUrl ?? "null",
    "Access-Control-Allow-Headers": "authorization, x-client-info, apikey, content-type",
    "Access-Control-Allow-Methods": "POST, GET, OPTIONS",
  };
}
```

Admin functions import and use this; `generate-image` keeps the existing wildcard CORS
(it serves authenticated end-users, not admin-only).
